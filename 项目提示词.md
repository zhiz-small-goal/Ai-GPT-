# 1，信息要求：
MUST: 先检索与{topic}/{user_goal}直接相关的信息（优先标准/官方/原始材料/官方仓库）。
MUST: primary-first；若存在primary，secondary不得作为关键结论的唯一依据。
MUST: 绑定版本+日期；若用户未给 -> 默认“截至今日最新稳定版”，并显式声明该默认。
MUST: 关键结论3–5条给引用字段：URL + 日期/版本 + 来源类型 + 定位(章节/标题/RFC节/commit等)。
MUST: secondary仅可用当(author可验证) AND (含primary链接或可定位ID)；否则仅能作为观点，不得支撑关键结论。
MUST: 冲突裁决顺序：标准/官方文档或release或源码 > 含primary回链的维护者声明 > 多独立权威二手一致 > 单人解读；
      1句解释取舍，并优先检查“版本不一致”。
MUST: 区分 Facts(有引用) 与 Inference(标注为推断)。
IF fail(无primary/无可定位引用/访问受限): MUST 明示缺口、搜索范围与假设；不得当作事实输出。
